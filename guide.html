Hermes agent guide

Simple guide to set up Hermes agents.

This is a clean starter guide for creating business-focused Hermes agents, including install, config, Telegram, and onboarding.

Back to dashboard
1. Install Hermes

Install Hermes on the machine or server where you want the agent to run.

curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
hermes doctor
2. Configure model and credentials

Pick the model/provider, then set the API credentials.

hermes setup
hermes model
hermes config path
hermes config env-path

Typical providers are OpenRouter, Anthropic, OpenAI, Gemini, and others supported by Hermes.

3. Set up gateway / Telegram

Connect the agent to Telegram so you can operate it from chat.

hermes gateway setup
hermes gateway install
hermes gateway start
hermes gateway status
  • Create the Telegram bot in BotFather
  • Add the bot token to Hermes config / env
  • Test the connection by messaging the bot
4. Create the business onboarding files

Every good business agent needs role, business knowledge, and guardrails.

SOUL.md
USER.md
knowledge/
  business-overview.md
  faq.md
  pricing.md
  policies.md
  workflows.md
  • SOUL.md: role, tone, constraints, priorities
  • USER.md: who the business owner is and how they work
  • knowledge docs: products, services, FAQs, policies, delivery, escalation
5. Recommended onboarding checklist
  1. Define the business goal for the agent
  2. Set the communication style
  3. Document products/services and FAQs
  4. Document what the agent must never guess or promise
  5. Add escalation rules for uncertain or sensitive cases
  6. Test the agent with real customer scenarios

7. Best first use cases for business agents
  • Customer enquiry assistant
  • Sales/admin assistant
  • Lead capture and follow-up assistant
  • Operations checklist assistant
  • Research and opportunity assistant

Start narrow. A focused agent becomes useful much faster than an “everything agent.”

8. Example onboarding rules
  • Do not invent stock, pricing, timelines, or policy decisions
  • If unsure, say you need to confirm
  • Escalate complaints, refunds, and unusual requests
  • Prefer concise summaries and actionable next steps
